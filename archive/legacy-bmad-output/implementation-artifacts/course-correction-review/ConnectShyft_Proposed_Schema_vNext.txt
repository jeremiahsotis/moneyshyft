ConnectShyft Proposed Schema vNext (2026-02-18)
Scope: greenfield schema for ConnectShyft on the updated Phase 0 platform (Tenant + OrgUnit +
BillingAccount). No prior code assumptions are required.
Notation: all timestamps stored as UTC. All phone numbers stored as E.164. Tables are Postgres-friendly;
adjust types if using another datastore.

C. Scoping and enforcement (locked)
Neighbor search is tenant-wide. Threads are orgUnit-scoped to the active orgUnit context.
Neighbor edits allowed only if user has an active thread with that neighbor in active orgUnit, or user has
tenant-privileged role.
Escalation clears only on claim. Outbound messages/calls do not clear escalation.
Prefers texting enforcement: enum UNKNOWN/YES/NO. If NO, outbound SMS requires override reason and
logs cs_sms_preference_overrides + audit event.
