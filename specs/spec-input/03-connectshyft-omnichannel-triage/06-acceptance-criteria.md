# Acceptance Criteria

## Channel support

- inbound phone events can be represented in triage
- SMS and MMS are represented as conversation-aware records
- email ingestion creates or appends to threads
- webchat sessions are stored and triaged
- website forms create structured triageable submissions

## Triage

- all unattached inbound items appear in one central queue
- queue can be filtered by status/channel/assignment
- intake staff can assign, escalate, resolve, and route items

## Linkage

- staff can search/create person or household from triage
- triage item can attach to existing case
- triage item can create new case
- triage item can attach to program or create program follow-up context

## Notifications

- assignment and escalation produce visible notifications for users/teams

## Auditability

- assignment, attachment, escalation, and resolution are recorded

## Safety

- existing ConnectShyft flows for currently supported channels are not broken
- rollout supports staged enablement for new channels if needed
