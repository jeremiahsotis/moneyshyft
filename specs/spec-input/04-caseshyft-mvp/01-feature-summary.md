# CaseShyft MVP — Feature Summary

## What is being built

CaseShyft MVP is the first operational case workspace for the ShyftUnity platform.

This feature introduces and formalizes:

- case creation
- case assignment
- case timeline
- case notes
- follow-up tasks
- basic service planning / service items
- person / household linkage
- embedded ConnectShyft communication context

CaseShyft is where triaged intake becomes actual ongoing work.

## Why now

After MoneyShyft PWA, shell foundations, PeopleCore, and ConnectShyft omnichannel triage, the next operational need is a structured workspace where staff can manage real cases.

Without CaseShyft:
- triaged intake has nowhere durable to live
- communications remain detached from work
- follow-up and assignment are ad hoc
- service coordination stays informal

CaseShyft MVP turns intake into managed support work.

## Who uses this

Primary users:
- case managers
- Vincentians
- intake staff escalating into case work
- supervisors

Secondary users:
- program managers viewing linked cases
- future coalition users
- administrative reviewers

## Success definition

A staff user should be able to:

1. create a case from triage or directly
2. link the case to a person or household
3. assign ownership
4. view a case timeline
5. add case notes
6. manage follow-up tasks
7. view and respond to linked communications
8. track basic service items / service planning steps
