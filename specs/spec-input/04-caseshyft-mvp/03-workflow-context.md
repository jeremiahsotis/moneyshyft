# Workflow Context

## A. Create case from triage

1. triage staff reviews inbound communication in ConnectShyft
2. staff links or creates person / household
3. staff selects "create case"
4. case is created with person / household linkage
5. communication is attached into case context
6. case opens in CaseShyft workspace

## B. Create case directly

1. user opens CaseShyft
2. user creates a new case
3. person / household is selected or created through PeopleCore integration
4. case metadata and initial note are entered
5. case is assigned or left unassigned

## C. Assignment workflow

1. case is assigned to a user or team
2. ownership is visible in workspace
3. reassignment is auditable

## D. Case note workflow

1. user opens case
2. user adds note
3. note appears in case timeline
4. note preserves author and timestamp

## E. Follow-up task workflow

1. user creates follow-up task from case
2. task is assigned and given status
3. task progress is tracked in case context

## F. Communication timeline workflow

1. case has linked communication thread(s) from ConnectShyft
2. communication history appears in case timeline or communication panel
3. user can reply or attach more communication where allowed
4. all activity stays tied to the case

## G. Service item / service plan workflow

1. user adds a service item or support action to the case
2. status is tracked
3. this creates a structured placeholder for later deeper resource/referral/finance integration

## H. Case close workflow

1. case reaches a natural stopping point
2. user closes case
3. history remains available
4. case stops appearing as active work
