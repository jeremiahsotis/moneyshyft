# Acceptance Criteria

## Case core

- case can be created directly
- case can be created from triage
- case links to person or household
- case assignment works

## Workspace

- case workspace displays summary and timeline
- notes can be added and viewed
- follow-up tasks can be added, assigned, and updated
- service items can be added and updated

## Communication integration

- linked communication history is visible in case context
- reply / attach actions are available where allowed
- communication linkage is auditable

## Lifecycle

- cases can move through open/active/on_hold/closed states
- case closure preserves history

## Safety

- CaseShyft does not duplicate PeopleCore or ConnectShyft ownership
- future documents/evidence/eligibility/referral integration remains possible without redesign

## Auditability

- case creation
- assignment changes
- note creation
- task creation/completion
- communication linkage
- case closure
are all recorded
