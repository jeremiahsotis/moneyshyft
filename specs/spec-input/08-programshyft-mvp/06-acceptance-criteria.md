# Acceptance Criteria

## Program core

- programs can be created and updated
- program status is tracked
- core program workspace is usable

## Sessions / cohorts

- sessions or cohorts can be configured where applicable
- lifecycle states are stored and queryable

## Participation

- participants can be linked through PeopleCore
- enrollment states can be tracked
- eligibility/document requirement visibility supports enrollment workflows

## Attendance / progress

- attendance can be recorded for session-based programs
- milestones or progress markers can be tracked

## Communication integration

- program-linked communication history is visible in program context
- reply / attach actions are available where allowed
- communication linkage is auditable

## Safety

- ProgramShyft does not duplicate ConnectShyft, PeopleCore, Eligibility, or Document/Evidence ownership
- future finance, referral, and deeper volunteer workflows remain possible without redesign

## Auditability

- program creation
- participant enrollment changes
- attendance recording
- milestone updates
- communication linkage
- program closure or archival
are all recorded
