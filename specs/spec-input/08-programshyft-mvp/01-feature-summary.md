# ProgramShyft MVP — Feature Summary

## What is being built

ProgramShyft MVP is the structured program operations workspace for ShyftUnity.

This feature introduces and formalizes:

- program records
- program configuration
- program participation
- cohorts and sessions
- attendance tracking
- milestones / progress markers
- program-linked communication context
- eligibility and document requirement visibility for enrollment
- program workflow actions inside the central shell

ProgramShyft is where recurring, cohort-based, event-based, and ongoing community programs are managed in a structured way.

## Why now

After ConnectShyft triage, CaseShyft, Documents + Evidence, Eligibility, and ResourceShyft, the next operational need is a structured environment for running programs.

Without ProgramShyft:
- programs are tracked in spreadsheets or disconnected systems
- enrollment and participation are inconsistent
- communications are detached from the actual program context
- eligibility and document requirements are not operationalized cleanly
- outcomes and attendance are harder to track

ProgramShyft turns community programs into a first-class operational workflow.

## Who uses this

Primary users:
- program managers
- program coordinators
- intake staff routing people into programs
- supervisors

Secondary users:
- case managers referring participants into programs
- volunteers supporting program operations
- future partner organizations

## Success definition

A staff user should be able to:

1. create and configure a program
2. define sessions or cohorts
3. enroll participants
4. see eligibility/document status relevant to enrollment
5. track attendance and progress
6. view and use program-linked communications
7. manage program work in a structured, repeatable way
