# Workflow Context

## A. Create program workflow

1. user creates a program
2. user enters core details, such as title, summary, audience, schedule style, and participation model
3. program is saved and becomes manageable in ProgramShyft

## B. Configure sessions / cohorts workflow

1. user defines whether the program runs as:
   - one-time event
   - recurring sessions
   - cohort-based program
   - ongoing opportunity
2. relevant session or cohort records are created
3. capacity or attendance structure is stored if applicable

## C. Enroll participant workflow

1. user starts enrollment from program context or receives a participant from case / resource flow
2. participant is linked through PeopleCore
3. eligibility/document/evidence status is shown if applicable
4. user records enrollment or pending enrollment outcome

## D. Attendance workflow

1. program session occurs
2. staff records attendance
3. participation state updates

## E. Milestone / progress workflow

1. program has milestones, checkpoints, or progress markers
2. staff updates progress
3. timeline / summary reflects the participant's current state

## F. Program communication workflow

1. program has linked communication context from ConnectShyft
2. staff views communications in program context
3. replies or follow-up actions are performed where allowed

## G. Program update / close workflow

1. program record changes over time
2. user updates configuration or marks program inactive/closed
3. history remains available for reporting and operations
