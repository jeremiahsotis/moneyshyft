# GitHub Issue Template Set for ShyftUnity

Use these templates to turn SpecKit output into consistent GitHub issues quickly.

Recommended flow:

1. Run SpecKit for a package.
2. Save output into the package folder:
   - `08-speckit-spec.md`
   - `09-speckit-task-plan.md`
   - `10-speckit-pr-slices.md`
3. Copy the appropriate template below.
4. Fill it from the SpecKit output.
5. Create GitHub issues in the same package order used in `/spec-input/RUN_ORDER.md`.

Template set included:

- `01-epic-template.md`
- `02-feature-issue-template.md`
- `03-pr-slice-issue-template.md`
- `04-bug-risk-hardening-template.md`
- `05-migration-template.md`
- `06-integration-test-template.md`
- `07-rollout-template.md`
- `08-issue-conversion-checklist.md`
