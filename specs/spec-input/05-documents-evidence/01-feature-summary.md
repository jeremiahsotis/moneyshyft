# Documents + Evidence Layer — Feature Summary

## What is being built

The Documents + Evidence Layer provides the shared verification foundation for ShyftUnity.

This work introduces and formalizes:

- document records
- document versions
- document requests
- participant upload links
- document intake and verification workflow
- document bundles for controlled sharing
- reusable evidence records
- evidence verification history
- evidence-to-document linkage
- evidence reuse across workflows

Documents and Evidence are closely connected, but they are **not the same thing**.

- A **document** is a file or artifact, such as an ID, utility bill, lease, pay stub, or signed release.
- **Evidence** is a fact or claim that may be supported by one or more documents, such as monthly income, county of residence, household size, or identity.

## Why now

After ConnectShyft triage and CaseShyft MVP, the next major operational pain is repeated document collection and the inability to reuse verified information.

Without this layer:
- people are asked for the same documents repeatedly
- staff cannot confidently reuse prior information
- future eligibility screening remains manual and inconsistent
- referrals cannot safely include the right supporting material
- document chaos continues across programs and services

## Who uses this

Primary users:
- case managers
- intake staff
- program managers
- document reviewers
- supervisors

Secondary users:
- participants uploading requested documents
- future referral recipients
- eligibility workflows
- resource/service workflows

## Success definition

A staff user should be able to:

1. request a document from a participant
2. send a secure upload link
3. receive the uploaded file
4. review and verify the document
5. link the document to reusable evidence
6. reuse that evidence later without re-collecting everything
7. include documents or bundles in future scoped workflows safely
