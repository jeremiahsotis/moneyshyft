# Acceptance Criteria

## Documents

- staff can create document requests
- participant upload links work
- direct staff upload works
- document records and versions are stored
- verification status is visible and auditable

## Evidence

- reusable evidence records can be created and updated
- evidence can link to one or more documents
- evidence verification history is preserved
- evidence usage can be recorded

## Reuse

- later workflows can distinguish reusable, needs confirmation, stale, and missing evidence
- worker can confirm unchanged evidence or update it

## Case integration

- document and evidence records can link to case context
- case users can understand whether a document is pending, verified, or rejected

## Safety

- Documents and Evidence remain separate but linked
- future Eligibility and Referral integration remains possible without redesign
- secure upload flows do not expose unrelated data

## Auditability

- document request, upload, verification, evidence update, and evidence reuse actions are recorded
