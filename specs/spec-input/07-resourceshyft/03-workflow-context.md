# Workflow Context

## A. Search workflow

1. user enters query or browses service categories
2. system returns ranked service offerings
3. user refines results using filters
4. user opens service detail

## B. Service detail workflow

1. user opens service offering
2. service detail shows:
   - provider
   - what the service does
   - who it serves
   - location or service area
   - availability / hours / current known access pattern
   - requirements / likely needed documents
3. user decides next action

## C. Eligibility-aware discovery workflow

1. user has a person/household/case context
2. ResourceShyft shows services that may fit
3. if eligibility result exists, service detail can show fit or missing requirement guidance
4. user launches next workflow

## D. Case launch workflow

1. user is in or coming from case context
2. selects a service
3. launches a case-linked service action or future referral

## E. Availability workflow

1. user asks what is open now or today
2. system surfaces currently open services first
3. then services available later that day
4. system explains timing in human-friendly language

## F. Admin/maintenance workflow

1. staff updates provider/service record
2. service metadata and availability are updated
3. freshness / verification information is preserved
4. service remains discoverable

## G. Future public-facing workflow

1. public user searches or browses
2. sees simplified service discovery experience
3. may transition into later contact or intake flow
